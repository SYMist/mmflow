# Money Flow Planner – 진행 내용 (2026-02-04 기준)

## 1. 문제 정의

- 월급 등 정기 수입이 여러 계좌/카테고리(생활비, 투자, 저축, 비상금 등)로 나누어 이체되는 구조를
  한 눈에 파악하기 어렵다는 문제.
- 사용자가 직접 “이상적인 흐름”을 설계하고, 각 노드 간 금액·비중을 시각적으로 관리할 수 있는 툴을 목표로 함.

## 2. v1(완전 수동) 제품 방향

- 은행 연동 없이, 사용자가 모든 노드·규칙을 직접 입력/편집.
- 플로우차트 UI(React Flow 기반)를 활용해 수입 → 목적지의 흐름을 수형도처럼 표시.
- 엣지(화살표)에는 **금액 + 비중(%)만** 라벨로 노출.
- 오른쪽 상세 패널에서 노드 이름 수정 및 삭제 가능.

## 3. 현재 UI 레이아웃 구조

- 상단 헤더 (`Header` – `src/components/layout/Header.tsx`)
  - 월 선택 (`<input type="month">`)
  - 시나리오 이름 입력
  - 이번 달 총 유입/이체/남는 돈 요약

- 본문 레이아웃 (`MainLayout` – `src/components/layout/MainLayout.tsx`)
  - 좌측: 사이드바 (`Sidebar`)
    - 수입 리스트 (`IncomeList`)
    - 목적지 리스트 (`DestinationList`)
    - 규칙 리스트 (`RuleList`) – “월급 → 생활비, ₩1,500,000 · 50%” 식 텍스트
  - 중앙: 플로우차트 캔버스 (`FlowCanvas`)
    - `@xyflow/react`(React Flow)로 노드/엣지 표시
    - 노드: 수입 / 목적지 노드
    - 엣지: 금액 + 비중(%)만 라벨로 표기
  - 우측: 상세 패널 (`DetailPanel`)
    - 노드 선택 시:
      - 항목 명 / 은행 이름 / 금액 입력 필드 (드래프트 편집 후 `확인` 버튼으로 일괄 적용)
      - 유형(수입/목적지), 삭제 버튼
    - 규칙 선택 시: 금액/비중 요약
  - 하단: 요약 (`BottomSummary`)
    - 목적지별 유입 합계 표

## 5. 구현된 기능 요약

- 초기 더미 데이터
  - `월급` 수입 노드, `생활비`, `투자` 목적지 노드
  - 월급 → 생활비 50%, 월급 → 투자 50% 규칙
- 노드 추가
  - 사이드바에서 수입/목적지 각각 `+ 추가` 버튼으로 새 노드 생성
- 노드 선택
  - 사이드바 리스트 또는 플로우캔버스에서 노드 클릭 시 오른쪽 상세 패널에 정보 표시
- 노드 필드 수정
  - 상세 패널에서 `항목 명 / 은행 이름 / 금액`을 입력한 뒤 `확인` 버튼을 눌러야 반영되도록 구현
- 노드 삭제
  - 상세 패널의 “노드 삭제” 버튼
  - 확인 다이얼로그 후:
    - 해당 노드 삭제
    - 해당 노드와 연결된 모든 엣지(규칙) 삭제
    - 선택 상태 초기화
- 엣지(규칙) 선택
  - React Flow에서 엣지 클릭 시 상세 패널에 금액/비중 요약
- 하단 요약
  - 목적지별 유입 합계를 계산해서 표로 출력

## 10. UX/상호작용 개선 (2026-02-05 업데이트)

- 노드 데이터 모델 확장
  - 각 노드에 `항목 명(itemName)`, `은행 이름(bankName)`, `금액(amount)` 저장
  - 캔버스 노드 카드, 사이드바 리스트, 하단 요약에서 모두 이 정보가 표시되도록 연결
- 상세 패널 편집 UX
  - 노드 선택 시:
    - 항목 명 / 은행 이름 / 금액을 드래프트로 편집
    - `확인` 버튼을 눌러야 실제 상태와 화면에 적용 (실수 방지)
- 캔버스 노드 표시/동작
  - 커스텀 React Flow 노드(`moneyNode`) 도입
    - 1줄: 항목 명
    - 2줄: 은행 이름(없으면 수입/목적지 텍스트)
    - 3줄: 금액(₩1,234,567 형식)
  - 노드 선택 시:
    - 더 강한 하이라이트(그라데이션 배경, 두꺼운 테두리, 깊은 그림자, 약간 확대)
  - 노드 드래그
    - 선택된 노드를 드래그하면 약간 투명해지고(`opacity ~0.7`), 커서가 `grabbing`으로 변경
    - 드래그 중(onNodeDrag)과 드래그 종료(onNodeDragStop) 시 `x, y`를 상태에 저장해 위치 유지
- 노드 주변 새 노드 생성(+ 버튼)
  - 각 노드 카드의 상/하/좌/우에 `+` 아이콘 표시 (hover 시 나타남)
  - 클릭 시:
    - 기준 노드가 `income`이면 → 해당 방향에 새 `destination` 노드 생성 + 기준 수입 → 새 목적지 엣지 추가
    - 기준 노드가 `destination`이면 → 새 `income` 노드 생성 + 새 수입 → 기준 목적지 엣지 추가
- 선택/캔버스 이동 동작
  - 노드/엣지 선택 해제:
    - 빈 캔버스 영역 클릭 시 `selectedNodeId`, `selectedEdgeId` 모두 해제
  - 드래그 방식:
    - 아무 것도 선택되지 않은 상태: 빈 캔버스를 왼쪽 드래그하면 전체 캔버스 이동
    - 노드 또는 엣지가 선택된 상태: 왼쪽 드래그는 노드 이동, 캔버스 이동은 오른쪽 버튼으로만 가능

## 11. 연결선/편집 UX 개선 (2026-02-06 업데이트)

- 연결 방향 고정
  - 각 노드에 방향별 핸들 ID 부여 (`s-top`, `s-right`, `s-bottom`, `s-left`, `t-top`, `t-right`, `t-bottom`, `t-left`)
  - 엣지 생성 시 `sourceHandleId`, `targetHandleId` 저장
  - 시작/도착 방향에 맞게 선이 해당 면에서 연결되도록 표시
- 연결선 드래그 연결
  - `+` 핸들에서 드래그하면 커서 따라 선이 나오고 다른 노드에 붙이면 연결 생성
- 연결선 재연결(시작점 변경)
  - 기존 엣지의 시작점을 끌어 다른 방향으로 놓으면 시작 핸들이 변경되도록 지원
  - `edgesReconnectable` 활성화 및 `onReconnect`에서 핸들 갱신
- 연결선 라벨 UX
  - 연결선 라벨(금액/비중)은 커스텀 엣지(`moneyEdge`)로 단일 렌더링
  - 라벨 위치는 곡선 중앙 기준으로 배치
  - 선택된 연결선에서만 금액 편집 가능
  - 선택된 연결선에 삭제(`×`) 버튼 노출
- 금액/비중 연동
  - 엣지 금액 수정 시 비중(%) 자동 재계산
  - 소스 노드 금액 변경 시 연결된 엣지 비중 재계산

## 12. 저장 로드 계획 (로컬 → 백엔드)

- 1단계: LocalStorage (다음 작업 범위)
  - 상태 스키마: `nodes`, `edges`, `scenarioName`, `month`, `updatedAt`
  - 자동 저장: 상태 변경 시 `localStorage`에 저장
  - 복원: 앱 시작 시 `localStorage`에서 불러오기
  - UX: “저장됨” 타임스탬프 또는 토스트 표시
- 2단계: 백엔드
  - Cloudflare Workers + D1 (SQLite) 추천
  - API 최소 스펙:
    - `GET /doc/:id` → 상태 반환
    - `POST /doc/:id` → 상태 저장
  - 로그인 없이 시작:
    - 브라우저에 `docId`(UUID) 생성 후 로컬에 저장
    - 동일 브라우저에서 동일 문서로 동기화
  - 충돌 대응:
    - `updatedAt` 기준 덮어쓰기(초기) → 이후 충돌 경고/머지로 확장
