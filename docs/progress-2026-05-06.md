# Money Flow Planner – 진행 내용 (2026-05-06 기준)

## 오늘의 주제: v3 안정화 — 기반 정비 + 그룹/영역 인터랙션 완성

## 1. 배경

전날(2026-05-05) v3 모델(노드 3종 + 그룹 + 영역) 도입을 PR로 머지한 후 사용자 검증 중 다음 문제들이 발견됨.

- Chrome에서 캔버스가 아예 렌더링되지 않음 (Safari는 정상)
- 그룹 선택이 작동하지 않음
- 영역 경계 리사이즈 시 비공유 영역의 노드가 사라지는 버그 (새로고침해야 보임)
- 노드/그룹의 "선택 상태"가 시각적으로 명확하지 않음
- 그룹 선택 후 위치 이동 불가

근본 원인: 오버레이/선택 로직이 빠르게 누적되며 이벤트 충돌, 이중 상태 관리, CSS 의존성 문제, z-index 미스가 한꺼번에 쌓임.

## 2. 기반 정비 (Option B 선택)

사용자와 협의 후 빠른 패치 대신 기반 재설계로 진행.

### 2.1 CSS 높이 체인 단순화
- `.app-root`: `min-height: 100vh` → `height: 100vh` + `html, body { overflow: hidden }`
- `.main-layout-v3`: 헤더 53px 가정(`calc(100vh - 53px)`) 제거, flex 단일 사용
- `.canvas-list-page`: `overflow-y: auto`로 자체 스크롤
- 결과: Chrome/Safari에서 동일하게 캔버스 렌더링

### 2.2 노드 상태 단일 소스
- `useNodesState` 제거 (React Flow 내부 상태 vs Zustand 스토어 이중 관리 문제 해소)
- `useMemo`로 store에서 직접 derive
- 드래그 중간 위치는 `draggingPositions` 로컬 state로 분리
- 외부에서 store가 변경돼도(예: 영역 리사이즈) 진행 중인 드래그와 충돌 없음
- → 영역 경계 리사이즈 시 노드 사라지는 버그 해결

### 2.3 선택 상태 통합
- `selectedNodeId`, `selectedEdgeId`, `selectedGroupId` 분산 → 단일 키로 통합:
  ```ts
  type Selection = 
    | { type: 'node'; id: string }
    | { type: 'edge'; id: string }
    | { type: 'group'; id: string }
    | null;
  ```
- 코드 단순화 + 동시 선택 불가 보장

### 2.4 z-index 구조 파악 + 정정
React Flow의 내부 z-index를 직접 조사:
- `.react-flow__pane`: z-index 1 (클릭 받는 투명 레이어)
- `.react-flow__viewport`: z-index 2
- `.react-flow__renderer` (노드/엣지): z-index 4
- `.react-flow__panel`: z-index 5

GroupOverlay/AreaOverlay가 zIndex 0이었기 때문에 라벨이 시각적으로는 보이지만 pane(z=1)이 위에 있어 클릭이 절대 도달하지 못한 상태였음. `onPaneClick`이 먼저 발동해 `clearSelection`이 즉시 호출되는 구조.

수정:
- GroupOverlay outer div: `zIndex: 10`
- AreaOverlay 핸들: `zIndex: 11`
- Body는 여전히 `pointer-events: none` (노드 클릭 통과)
- 라벨/핸들에 `nodrag nopan` 클래스 + 모든 이벤트 `stopPropagation`

## 3. UX 개선

### 3.1 클릭-선택 패턴
- 노드: `draggable: !isEditing && node.type !== 'sub' && isSelected` — 선택된 노드만 드래그 가능
- 그룹: 라벨 클릭으로 선택, 선택 시에만 리사이즈/이동 핸들 활성화
- 두 단계로 분리해 의도치 않은 이동 방지

### 3.2 선택 상태 시각 표시
- **노드 선택 시**: 보더 1.5px → 2px (primary 색), 외곽 보라색 발광(3px shadow ring), 점선 outline(6px 바깥), 상단에 "선택됨 · 드래그로 이동" 뱃지, cursor: move
- **그룹 선택 시**: 점선 보더 1.5px → 2.5px, 외곽 보라색 4px 발광 + 그림자, 라벨 색상 반전(흰 배경/보라 글자 → 보라 배경/흰 글자), "그룹 선택됨 · 라벨 드래그로 이동 · 모서리 드래그로 크기 조정" 뱃지

### 3.3 영역 리사이즈 시 노드 함께 이동
- `setAreaBounds`가 좌/우 영역 노드들의 x좌표를 boundary 변화량만큼 자동 시프트
- 비공유 노드가 의도치 않게 공유 영역으로 흘러들어가지 않음
- 좌측 영역 확장 → 그 안의 좌측 노드들도 함께 좌로 이동

### 3.4 그룹 이동 (신규)
- 선택된 그룹의 라벨을 드래그하면 그룹 전체 이동
- `moveGroup(groupId, dx, dy)` 액션: 멤버 도착 노드 + 그 서브 노드 + 그룹 explicit bounds 모두 dx/dy 시프트
- 드래그 임계값 3px로 의도치 않은 미세 이동 방지

### 3.5 그룹 박스 밖으로 노드 드롭 시 자동 탈퇴
- `tryGroupOnDrop`에서 그룹 멤버 노드가 그룹 박스 밖으로 떨어지면 자동으로 `removeFromGroup`
- 명시적 bounds가 있으면 그 박스 기준, 없으면 다른 멤버들 bounding box 기준
