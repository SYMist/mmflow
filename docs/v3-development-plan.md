## 모델 v3 핵심

### 노드
| 타입 | 설명 | 부모 | 자식 |
|------|------|------|------|
| 출발 노드 | 외부에서 들어온 돈 (월급 등) | 없음 | 도착 노드 N개 |
| 도착 노드 | 돈이 모이는 계좌 | 출발/서브 1개 (1:1 강제) | 서브 노드 0~1개 |
| 서브 노드 | 도착 노드의 자체 수익 (배당, 이자) | 도착 노드 1개 (종속) | 도착 노드 N개 |

- 모든 노드 필드: `name`, `account`, `amount` (사용자 직접 입력)
- 엣지에는 amount/ratio 없음 — 도착 노드의 amount가 곧 흐름량
- 서브 노드는 부모 도착 노드 옆에 붙어서 함께 이동

### 그룹
- 동일 `name + account`를 갖는 **도착 노드들**만 그룹 가능
- 그룹 형성 액션:
  - (a) 두 도착 노드를 드래그앤드롭으로 겹침
  - (b) 1:1 위반(2번째 부모 연결 시) → 자동 복제 + 그룹 제안 모달 → 사용자 수락
- 한 도착 노드는 정확히 1개 그룹에만 속함 (그룹 미참여도 가능)
- 그룹 합계 = 멤버들의 amount 합 (자동 계산, 별도 목표 amount 없음)
- 시각: 점선 박스 + 합산 라벨

### 영역 (멀티 멤버 캔버스 한정)
- 캔버스 구성원 수가 2 이상일 때 각 멤버는 영역(점선 박스)을 가짐
- 노드 속성: `memberId` (소유 영역), `isShared` (공유 여부)
- 비공유 노드는 타 영역 노드와 연결 불가
- 영역 경계로 드래그 → 공유 노드로 전환
- 비공유 도착 노드끼리 그룹이 영역을 가로지르면 → 공유 자동 전환
- 줌은 단일 (영역별 독립 줌은 v3 보류)

### 캔버스
| 필드 | 설명 |
|------|------|
| name | 캔버스 이름 |
| memberType | '가족' \| '친구' \| '연인' \| '배우자' |
| memberCount | 정수 (연인/배우자는 2 고정) |
| memberNames | 멤버 이름 배열 (memberCount만큼) |
| goal | '가구 경제 관리' \| '데이트 통장 관리' \| ... |
| createdAt | 생성일 |

- 구성원 추가/삭제는 생성 후 불가
- 월별로 캔버스 안의 노드/엣지가 분리 저장 (현재와 동일)

### 합산 노출
- 출발 노드 + 서브 노드의 amount 합산
- 단일 멤버: 전체 합 1개
- 멀티 멤버: 공유 영역 합 + 영역별 합

### 경고 규칙
- 출발/서브 노드의 amount < 직접 자식 도착들의 amount 합 → 노드/그룹에 적색 표시
- 충족되면 자동 해제

## 폐기 / 재사용

### 폐기 대상 (v2 → 미사용)
- `'income' | 'destination'` 노드 타입
- 엣지 amount/ratio 입력 UI (`MoneyEdge.tsx`의 인라인 편집)
- 시나리오 ↔ 단순 dropdown 전환 UI (캔버스 리스트로 대체)
- v2 localStorage 키 (`mf:scenarios`, `mf:{scenarioId}:{month}`) — 새 prefix `mfv3:` 도입, 기존은 무시

### 재사용 가능
- React Flow 캔버스 골격
- Zustand 스토어 패턴 + Undo/Redo + persist 디바운스
- 인라인 노드 편집 컴포넌트 (FlowNode 편집 모드)
- 다크모드 + 디자인 토큰
- JSON export/import 골격
- 월 피커 컴포넌트 (캔버스별 데이터 유무 표시 로직만 v3 키로 변경)
- 키보드 단축키 (Ctrl+Z, Delete 등)
- EmptyState 패턴
